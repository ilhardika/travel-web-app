Proses Bisnis Aplikasi Travel Booking
1. Proses user
- Mulai (Start): Proses dimulai ketika user mengunjungi aplikasi.
- User Visits Landing Page: user membuka halaman utama aplikasi, di mana mereka dapat melihat berbagai aktivitas yang ditawarkan.
- User Browses Activities: user menjelajahi berbagai aktivitas yang tersedia.
- User Views Activity Details: user memilih aktivitas tertentu untuk melihat detail lebih lanjut.
- Is User Logged In?: Sistem memeriksa apakah user sudah login.
  - Jika Ya: user dapat langsung menambahkan aktivitas ke keranjang.
  - Jika Tidak: user diarahkan untuk melakukan login atau registrasi.
    - Login Successful: Jika login berhasil, user dapat menambahkan aktivitas ke keranjang.
- User Adds Activity to Cart: user menambahkan aktivitas yang dipilih ke dalam keranjang belanja.
- User Views Cart: user melihat isi keranjang belanja mereka.
- User Proceeds to Payment: user melanjutkan ke proses pembayaran.
- User Uploads Payment Proof: user mengunggah bukti pembayaran untuk transaksi.
- System Confirms Payment: Sistem memverifikasi dan mengonfirmasi pembayaran yang telah dilakukan.
- User Views Transaction History: user dapat melihat riwayat transaksi mereka.
- Select Action: user memilih tindakan selanjutnya:
  - View Details: user dapat melihat detail transaksi tertentu.
  - Cancel Order: Jika user memilih untuk membatalkan pesanan, sistem akan memperbarui status pesanan.

2. Proses Admin
- Admin Logs In: Admin melakukan login ke sistem untuk mengakses fitur manajemen.
- Admin Accesses Dashboard: Setelah login, admin mengakses dashboard untuk melihat berbagai opsi manajemen.
- Select Management Option: Admin memilih opsi manajemen yang diinginkan, seperti:
  - Manage Banners: Mengelola banner promosi di halaman utama.
  - Manage Promotions: Mengelola promosi yang sedang berlangsung.
  - Manage Categories: Mengelola kategori aktivitas.
  - Manage Activities: Mengelola aktivitas yang ditawarkan.
  - Manage Users: Mengelola data user.
  - Manage Transactions: Mengelola transaksi yang dilakukan oleh user.
- Select Transaction Action: Jika admin memilih untuk mengelola transaksi, mereka dapat:
  - Update Status: Memperbarui status transaksi tertentu.
  - Cancel Transaction: Membatalkan transaksi yang diinginkan.
    - System Updates Transaction Status: Sistem akan memperbarui status transaksi sesuai dengan tindakan yang diambil oleh admin.

Kesimpulan
Proses bisnis ini menggambarkan interaksi antara user dan admin dalam aplikasi Travel Booking. user dapat menjelajahi, memilih, dan membayar aktivitas, sementara admin memiliki kontrol untuk mengelola konten dan transaksi dalam sistem. Proses ini dirancang untuk memberikan pengalaman user yang lancar dan efisien, serta memastikan bahwa admin dapat mengelola sistem dengan baik.
